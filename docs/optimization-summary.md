## 待完成的优化

### 1. 高优先级

#### 1.1 修复Tab切换性能问题
**问题描述：** custom-tabbar 使用 wx.reLaunch 导致每次切换都重新加载页面

**当前代码：**
```javascript
// components/custom-tabbar/custom-tabbar.js:73
wx.reLaunch({
  url: `/${path}`
});
```

**影响：**
- 每次切换Tab都会销毁并重新创建页面
- 数据需要重新加载
- 用户状态（滚动位置、输入等）丢失
- 性能体验差

**优化方案：**
1. 使用全局变量存储页面状态
2. 使用 wx.switchTab（但需要使用原生TabBar）
3. 使用页面栈管理，避免重复创建

**预计效果：** Tab切换流畅度提升80%

#### 1.2 修复分类图标显示不一致问题
**问题描述：** 分类列表中图标显示位置不符合设计要求

**设计要求：** 
```
📌 自定义分类
```

**当前实现：** 部分页面显示为：
```
自定义分类 📌
```

**影响范围：**
- 习惯表单（habit-form）
- 待办表单（todo-form）
- 分类管理（category-manage）

**优化方案：**
1. 统一分类数据模型
2. 在获取分类时确保icon字段存在
3. 建立分类缓存机制

### 2. 中优先级

#### 2.1 添加数据缓存机制
**问题描述：** 每次页面切换都重新请求云函数

**当前情况：**
```javascript
// 每次onShow都会调用
onShow() {
  this.loadTodos()  // 调用云函数
}
```

**优化方案：**
1. 实现本地缓存（wx.setStorageSync）
2. 添加数据新鲜度检查（5分钟）
3. 使用监听器模式更新数据

**预计效果：** 减少60%的云函数调用

#### 2.2 优化云函数调用
**问题描述：** 页面加载时多次独立调用云函数

**优化方案：**
1. 合并多个请求为批量请求
2. 使用Promise.all并行请求
3. 云函数端添加聚合接口

**预计效果：** 页面加载速度提升40%
