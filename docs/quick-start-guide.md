# 快速开始指南

## 🚀 最快的上手方式（推荐）

### 方式一：使用数据库初始化页面（最简单）

只需点击一个按钮，自动完成所有初始化！

#### 步骤：
1. **部署 initDB 云函数**
   ```
   右键 cloudfunctions/initDB → 上传并部署：云端安装依赖
   ```

2. **打开数据库初始化页面**
   - 在浏览器中添加页面参数：
   ```
   http://你的小程序地址/pages/database-init/database-init
   ```
   或者在微信开发者工具中：
   - 在 `app.json` 中将 `database-init` 放在 `pages` 数组的第一位（临时）
   - 重新编译后，这是首页
   - 点击"初始化数据库"按钮

3. **完成初始化**
   - 点击"初始化数据库"按钮
   - 等待几秒钟
   - 看到"初始化完成"提示即可

4. **恢复正常首页**
   - 在 `app.json` 中将 `pages/index/index` 放回第一位
   - 重新编译

## 📋 完整部署流程（第一次使用）

### 第 1 步：开通云开发（必须）

1. 在微信开发者工具中，点击顶部菜单栏的 **"云开发"**
2. 如果提示开通，点击 **"开通"**
3. 选择环境：`zcsimple1-yun001`
4. 创建环境（如果还没有）

### 第 2 步：部署云函数（按顺序）

**必须先部署 common：**
```
右键 cloudfunctions/common → 上传并部署：云端安装依赖
```

**然后部署其他云函数：**
```
右键 cloudfunctions/initDB → 上传并部署：云端安装依赖
右键 cloudfunctions → 上传并部署：云端安装依赖
```

### 第 3 步：使用初始化页面

1. 编译小程序
2. 在 app.json 的 pages 数组中，临时将：
   ```json
   "pages/database-init/database-init"
   ```
   移到第一行
3. 重新编译
4. 现在首页就是初始化页面
5. 点击 **"初始化数据库"** 按钮
6. 等待完成（大约 2-3 秒）
7. 完成后，将 `pages/index/index` 放回第一行
8. 重新编译

### 第 4 步：开始使用

现在可以：
- ✅ 创建习惯
- ✅ 添加待办
- ✅ 查看统计
- ✅ 管理分类

## 📚 云函数部署顺序（重要）

```
1. common         ← 必须最先部署
2. initDB         ← 数据库初始化
3. user           ← 用户管理
4. category       ← 分类管理
5. habit          ← 习惯管理
6. checkin        ← 打卡功能
7. todos          ← 待办管理
8. stats          ← 统计功能
9. login          ← 登录（可选）
```

**为什么顺序很重要？**
- `user`, `category`, `habit`, `checkin`, `todos`, `stats` 都依赖 `common`
- 如果先部署这些会报错，因为找不到 `common` 模块

## ⚡ 快速故障排除

### 问题 1：页面显示空白

**原因：** 云函数未部署  
**解决：**
```
右键 cloudfunctions → 上传并部署：云端安装依赖
```

### 问题 2：初始化失败

**原因：** users 集合不存在  
**解决：**
- 先手动在云开发控制台创建 `users` 集合
- 再点击初始化按钮

### 问题 3：预设分类重复

**原因：** 多次点击初始化  
**解决：** 正常现象，已存在的会自动跳过

### 问题 4：所有功能都超时

**原因：** 云函数超时时间太短  
**解决：**
1. 打开云开发控制台 → 云函数
2. 点击对应的云函数
3. 点击"配置"
4. 超时时间设置为 60 秒
