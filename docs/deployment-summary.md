## 🔧 云函数部署顺序（重要）

```
第 1 步：common（必须最先）
├── 原因：其他云函数依赖 common 模块
└── 操作：右键 cloudfunctions/common → 上传并部署：云端安装依赖

第 2 步：initDB（数据库初始化）
├── 功能：一键创建用户记录和导入预设分类
└── 操作：右键 cloudfunctions/initDB → 上传并部署：云端安装依赖

第 3 步：其他云函数（批量）
├── 包括：user, category, habit, checkin, todos, stats
└── 操作：右键 cloudfunctions → 上传并部署：云端安装依赖
```

**注意事项：**
- 必须先部署 common，否则其他云函数会报错
- 批量部署时，如果某个失败，单独重新部署即可
- 部署过程中不要关闭开发者工具

**action: init** - 完整初始化
- 创建用户记录（users 集合）
- 导入 9 个预设分类（categories 集合）
- 自动设置用户偏好

**action: addPresetCategories** - 仅添加预设分类
- 检查是否已存在
- 跳过已存在的
- 添加新的分类

**action: getStatus** - 检查数据库状态
- 检查 5 个集合是否创建
- 返回每个集合的数据条数
- 用于初始化页面显示状态
