# Bug修复记录 - 2026年3月31日

## 问题概述

用户反馈：添加习惯页面分类选择器中，分类图标不显示，只显示分类名称。这是之前已修复问题的回归。

## 根本原因

在之前的修改中，我虽然修改了`habit-form.wxml`将picker的range-key从"name"改为"displayName"，但是**忘记在`habit-form.js`的`loadCategories`方法中为每个分类添加`displayName`属性**。

这导致：
1. picker尝试使用不存在的`displayName`字段
2. 回退到显示对象或undefined
3. 最终只显示分类名称，没有图标

## 修复内容

### 文件1: miniprogram/pages/habit-form/habit-form.js

#### 修复1: loadCategories方法
**问题**: 没有为分类添加displayName属性

**修复前**:
```javascript
const allCategories = getAllCategories(customCategories)

this.setData({
  categories: allCategories
})
```

**修复后**:
```javascript
// 合并预设分类和自定义分类，添加displayName用于picker显示
const allCategories = getAllCategories(customCategories).map(cat => ({
  ...cat,
  displayName: `${cat.icon} ${cat.name}`
}))

this.setData({
  categories: allCategories
})
```

#### 修复2: updateCategoryInfo方法
**问题**: 使用`category.name`而不是`category.displayName`

**修复前**:
```javascript
this.setData({
  categoryIndex: index >= 0 ? index : 0,
  categoryName: category ? category.name : '选择分类'
})
```

**修复后**:
```javascript
this.setData({
  categoryIndex: index >= 0 ? index : 0,
  categoryName: category ? category.displayName : '选择分类'
})
```

#### 修复3: updateCategoryInfo方法中的分类匹配逻辑
**问题**: 只检查`c._id`，没有考虑预设分类的`id`字段

**修复前**:
```javascript
const index = categories.findIndex(c => c._id === categoryId)
```

**修复后**:
```javascript
const index = categories.findIndex(c => c._id === categoryId || (c.isPreset && c.id === categoryId))
```

#### 修复4: onCategoryChange方法中的categoryId获取
**问题**: 只获取`category._id`，没有考虑预设分类的`id`字段

**修复前**:
```javascript
this.setData({
  categoryId: category ? category._id : '',
  categoryIndex: index,
  categoryName: category ? category.displayName : '选择分类'
})
```

**修复后**:
```javascript
this.setData({
  categoryId: category ? (category._id || category.id) : '',
  categoryIndex: index,
  categoryName: category ? category.displayName : '选择分类'
})
```

### 文件2: miniprogram/pages/index/index.wxml

#### 修复: habit-item的class绑定
**问题**: bindtap使用了字符串而不是动态绑定

**修复前**:
```xml
<view class="habit-item {{habit.checked ? 'hidden-item' : ''}} {{!showCompleted && habit.checked ? 'show-hidden' : ''}}"
```

**修复后**:
```xml
<view class="habit-item {{habit.checked ? 'checked' : ''}} {{!showCompleted && habit.checked ? 'hidden-item' : ''}}"
```

## 预防措施

### 1. 代码审查清单
在进行涉及数据结构的修改时，确保：
- [ ] 修改wxml的range-key时，同步修改JS中的数据结构
- [ ] 涉及预设分类和自定义分类混用时，同时检查`_id`和`id`字段
- [ ] 修改字段名时，搜索所有使用该字段的地方

### 2. 测试检查点
每次修改后，必须测试：
- [ ] 表单的输入、回显、保存
- [ ] picker/ActionSheet的显示和选择
- [ ] 列表的渲染和交互
- [ ] 数据的完整性和一致性

### 3. 日志和调试
在关键位置添加console.log：
```javascript
loadCategories() {

  console.log('Loaded categories:', allCategories.map(c => ({
    name: c.name,
    displayName: c.displayName,
    _id: c._id,
    id: c.id
  })))
}
```
