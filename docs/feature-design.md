# 功能设计方案

## 1. 习惯管理功能

### 1.1 功能入口
**首页** - 右上角添加管理按钮
- 位置：与"今日打卡"标题在同一行
- 样式：圆形按钮，图标为 ⚙️ 或 📋
- 点击后跳转到习惯管理页面

### 1.2 习惯管理页面功能

#### 功能列表
- **查看所有习惯**：展示用户创建的所有习惯列表
- **编辑习惯**：修改习惯的名称、时间、分类等
- **删除习惯**：删除不再需要的习惯
- **调整顺序**：拖拽或点击按钮调整习惯显示顺序
- **启用/禁用**：临时禁用某个习惯，不删除但也不显示
- **批量操作**：支持批量删除、批量禁用等

#### 界面设计
```
习惯管理
─────────────────────────
📋 早上锻炼  每天 08:00
   健康 | ✅ 已启用

📝 阅读书籍  每天 20:00
   学习 | ✅ 已启用

🎸 练习吉他  每天 19:00
   成长 | ❌ 已禁用

+ 添加新习惯
─────────────────────────
```

### 1.3 技术实现
- 新建页面：`pages/habit-manage/habit-manage`
- 云函数：`habit` - 添加 `listAll`, `update`, `delete`, `disable`, `enable`, `reorder` 操作
- 数据库：`habits` 集合，添加 `disabled` 字段

## 2. 待办功能优化

### 2.1 未来日期待办支持

#### 当前限制
- 日期选择器只能选择今天或过去的日期
- 未来日期的待办无法添加

#### 优化后功能
- **日期选择**：可以选择任意日期（包括未来日期）
- **未来待办**：
  - 可以添加未来日期的待办
  - 未来待办显示为灰色，不可点击完成
  - 显示提示"这是未来的待办"
- **当前/过去待办**：
  - 正常显示
  - 可以点击完成
  - 已完成的可以隐藏

### 2.2 待办页面设计

#### 日期选择器区域
```
◀  2026年3月31日  ▶
   星期二  今天
```

#### 未来日期提示
```
─────────────────────────
ℹ️ 这是未来的待办
日期选择器可以切换到当天或过去日期
来完成待办事项
─────────────────────────
```

#### 待办列表样式
```javascript
// 未来的待办
<view class="todo-item future-todo">
  <view class="todo-checkbox" disabled>☐</view>
  <view class="todo-content">
    <text class="todo-title">准备会议材料</text>
    <text class="todo-time">10:00</text>
  </view>
</view>

// 当天/过去的待办
<view class="todo-item">
  <view class="todo-checkbox" bindtap="completeTodo">☐</view>
  <view class="todo-content">
    <text class="todo-title">完成报告</text>
    <text class="todo-time">14:00</text>
  </view>
</view>
```

### 2.3 技术实现
- 修改 `pages/todos/todos.js`：
  - `onPrevDay()`, `onNextDay()` - 支持切换到未来日期
  - `updateDateDisplay()` - 判断是否是未来日期，显示提示
  - `checkIsFuture()` - 判断当前选择日期是否是未来日期
- 修改云函数 `todo`：
  - 未来日期的待办可以正常添加和查询
  - 状态管理区分：`pending`, `completed`, `future`

## 7. UI 设计规范

### 按钮样式
- **主按钮**：紫色渐变背景 `linear-gradient(135deg, #667eea 0%, #764ba2 100%)`
- **次按钮**：白色背景，紫色文字
- **危险按钮**：红色背景 `#ef4444`

### 列表项样式
- **高度**：100rpx
- **间距**：12rpx
- **圆角**：16rpx
- **阴影**：`0 2rpx 8rpx rgba(0, 0, 0, 0.05)`

### 提示信息样式
- **成功**：绿色背景 `#10b981`
- **警告**：黄色背景 `#f59e0b`
- **错误**：红色背景 `#ef4444`
- **信息**：蓝色背景 `#3b82f6`

## 8. 注意事项

### 用户体验
1. **确认操作**：删除、清空数据等危险操作需要二次确认
2. **加载状态**：数据加载时显示加载动画
3. **空状态**：无数据时显示友好提示
4. **错误提示**：操作失败时显示清晰的错误信息

### 性能优化
1. **分页加载**：习惯列表较多时使用分页
2. **缓存策略**：频繁使用的数据进行缓存
3. **懒加载**：图片等资源按需加载

### 数据安全
1. **备份机制**：定期提示用户备份数据
2. **数据验证**：恢复数据时进行格式验证
3. **权限控制**：确保用户只能访问自己的数据
